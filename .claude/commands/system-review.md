# /system-review

**Model:** Opus
**Mode:** Fast → escalate to Critical if security surfaces found

Perform a full system reading:
1. map architecture from evidence — modules, boundaries, flows, integrations
2. map critical flows — auth, billing, publish, deploy, migrations
3. identify top 10 risks: structural, operational, security
4. propose phased roadmap — value × risk × reversibility
5. recommend next minimum safe step

**Rules:**
- do not edit before the plan is presented and approved
- base all conclusions on repository evidence — no assumptions
- distinguish: evidence / inference / decision
- if evidence is insufficient, say so explicitly
