# Engineering Governance Policy

## Change philosophy
All meaningful changes must be:
- justified
- minimal
- reversible
- validated
- documented when material

## Required before implementation
- context review
- impact review
- risk review
- validation plan
- rollback plan

## Required after implementation
- summary of edits
- validation evidence
- residual risk statement
- rollback note

## Forbidden without approval
- production-impacting operations
- destructive edits
- irreversible data changes
- secret mutations
- major dependency churn
- interface-breaking changes
