# Git recovery (docs entry)

Canonical policy lives at the repository root: [`GIT-RECOVERY.md`](../GIT-RECOVERY.md).

Use that file for fetch-first, rebase conflicts, nested clones, stash discipline, and forbidden commands. For nested `claude-operating-system/`, root `GIT-RECOVERY.md` documents **WARN** (default `verify-git-hygiene.ps1`) vs **FAIL** (`-Strict` or CI).
